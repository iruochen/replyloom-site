REPLYLOOM

Privacy Policy

Effective June 23, 2026

ReplyLoom is a browser extension that helps users draft replies to posts on X. It does not publish replies automatically, operate an engagement bot, or run a developer-owned model proxy.

Data ReplyLoom handles

  • The text and public context of the X post selected by the user.
  • Reply preferences such as language, tone, length, and optional writing instructions.
  • The API key, provider, endpoint, and model selected by the user.
  • Local interface preferences and floating-panel positions.

How data is used

When the user requests reply drafts, the selected post context and reply preferences are sent directly from the extension to the model provider selected by the user. The API key is sent only to that provider for authentication.

Supported providers include OpenAI, DeepSeek, MiniMax, Zhipu GLM, Volcengine Ark (Doubao), Alibaba Cloud Model Studio (Qwen), Moonshot AI, and SiliconFlow. Each provider processes requests under its own terms and privacy policy.

Storage and retention

  • API keys are stored in Chrome extension storage only when the user enables the save-key option. Otherwise they remain in session storage.
  • Provider settings and interface preferences are stored locally.
  • Selected post context is kept in session storage for the current browser session.
  • ReplyLoom does not send API keys, post content, or generated replies to a server operated by the ReplyLoom developer.

Users can remove locally stored data by clearing the extension's data or uninstalling ReplyLoom.

Data sharing

ReplyLoom shares selected post context and generation instructions only with the model provider chosen by the user. It does not sell personal data, use data for advertising, perform creditworthiness checks, or share data with unrelated third parties.

Permissions and security

ReplyLoom accesses X pages only to read the post selected by the user and insert a user-approved draft into the relevant reply editor. Network access is limited to an explicit list of supported HTTPS model-provider domains. ReplyLoom does not execute remotely hosted code.

User control

ReplyLoom creates editable drafts. The user must review and manually publish every reply. The extension does not click X's final Reply or Post button.
